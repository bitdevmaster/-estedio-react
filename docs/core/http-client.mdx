## Overview

The HTTP runtime is built on Axios and shared by every adapter.

| Component         | Responsibility                                               |
| ----------------- | ------------------------------------------------------------ |
| `apiClient`       | Auth headers, 401 refresh handling, normalized errors        |
| `ResourceAdapter` | Maps `ResourceConfig` to an HTTP call                        |
| `RequestQueue`    | Queues pending requests while a token refresh is in progress |

## ResourceAdapter

`ResourceAdapter` implements `IApiPort` and turns a resource name into an HTTP request.

```ts
import { ResourceAdapter } from "@estedio/react";

const adapter = new ResourceAdapter(resources);
const users = await adapter.execute("GetUsers");
```

### Path parameters

```ts
adapter.execute("GetUserById", {
  userId: 7,
  includePosts: true,
});
```

For an endpoint like `/users/:userId`, the adapter interpolates `userId` into the path and forwards the remaining payload values as the request body or query params.

## apiClient

`apiClient` handles:

- Resolving `apiBaseUrl` and `refreshEndpoint` from `LibraryConfig`
- Adding the `Authorization` header unless `skipAuth` is set
- Refreshing tokens on `401` once per request
- Normalizing thrown errors into `NormalizedApiError`

### skipAuth

```ts
await apiClient.get("/public/health", undefined, { skipAuth: true });
```

### NormalizedApiError

```ts
interface NormalizedApiError {
  status: number;
  code: string;
  ray_id: string;
  message: string;
}
```

Use `normalizeError` and `isNormalizedApiError` from the root package when you need to align application-level error handling with the library runtime.

## RequestQueue

`RequestQueue` is specifically used during token refresh. If multiple authenticated requests fail with `401` while a refresh is already running, later requests wait for that refresh and retry after it succeeds.

## Resource metadata used by the HTTP and state layers

| Field                 | Description                                        |
| --------------------- | -------------------------------------------------- |
| `authenticated`       | Controls whether the auth header is attached       |
| `meta.persist`        | Enables persisted response storage                 |
| `meta.persistKey`     | Storage key for persisted hydration                |
| `meta.isAuthResource` | Saves `access_token` and `refresh_token` responses |
| `sagaEffect`          | Redux-only watcher strategy                        |
