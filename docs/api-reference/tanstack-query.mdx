## createResources

```ts
function createResources<TResourceMap>(
  resources: Record<keyof TResourceMap, ResourceConfig>,
): TanstackResourceHooks<TResourceMap>;
```

The main factory function for the TanStack Query adapter. Returns `useResourceQuery` and `useResourceMutation` hooks typed to your `ResourceTypeMap`.

No `options` parameter â€” cache control is handled by the `QueryClient` you provide.

## createResourceHooks

```ts
function createResourceHooks<TResourceMap>(
  resources: Record<keyof TResourceMap, ResourceConfig>,
): TanstackResourceHooks<TResourceMap>;
```

Alias of `createResources`. Both functions are identical.

## useResourceQuery

```ts
function useResourceQuery<K extends keyof TResourceMap>(params: {
  resource: K;
  payload?: TResourceMap[K]["payload"];
  force?: boolean;
}): QueryResourceResult<TResourceMap[K]["response"]>;
```

Executes a GET-style resource as a TanStack Query `useQuery` call.

```ts
const { result, loading, error } = useResourceQuery({
  resource: "GetUsers",
  force: false,
});
```

### QueryResourceResult

```ts
interface QueryResourceResult<T> {
  result: T | null;
  loading: boolean;
  error: string | null;
}
```

| Field     | Type             | Description                               |
| --------- | ---------------- | ----------------------------------------- |
| `result`  | `T \| null`      | API response data when the query succeeds |
| `loading` | `boolean`        | `true` while the query is in-flight       |
| `error`   | `string \| null` | Error message if the query failed         |

## useResourceMutation

```ts
function useResourceMutation<K extends keyof TResourceMap>(
  resource: K,
): MutationResourceResult<
  TResourceMap[K]["payload"],
  TResourceMap[K]["response"]
>;
```

Wraps a resource as a TanStack Query `useMutation` call.

```ts
const createUser = useResourceMutation("CreateUser");
createUser.dispatch({ name: "Alice", email: "alice@example.com" });
```

### MutationResourceResult

```ts
interface MutationResourceResult<TPayload, TResponse> {
  dispatch: (payload: TPayload) => void;
  result: TResponse | null;
  loading: boolean;
  error: string | null;
}
```

| Field      | Type                          | Description                            |
| ---------- | ----------------------------- | -------------------------------------- |
| `dispatch` | `(payload: TPayload) => void` | Triggers the mutation                  |
| `result`   | `TResponse \| null`           | Last successful mutation response      |
| `loading`  | `boolean`                     | `true` while the mutation is in-flight |
| `error`    | `string \| null`              | Error message if the mutation failed   |

## Types

### TanstackResourceHooks

```ts
interface TanstackResourceHooks<TResourceMap> {
  useResourceQuery: <K extends keyof TResourceMap>(params: {
    resource: K;
    payload?: TResourceMap[K]["payload"];
    force?: boolean;
  }) => QueryResourceResult<TResourceMap[K]["response"]>;

  useResourceMutation: <K extends keyof TResourceMap>(
    resource: K,
  ) => MutationResourceResult<
    TResourceMap[K]["payload"],
    TResourceMap[K]["response"]
  >;
}
```

### FetchCallback

Internal async fetch function type:

```ts
type FetchCallback = (payload?: unknown) => Promise<unknown>;
```

## Notes

- `meta.persist` and `meta.isAuthResource` are **not** supported by this adapter.
- `sagaEffect` is a Redux-only field and is ignored.
- For caching, configure `staleTime` and `gcTime` on the `QueryClient` directly.
- You must wrap your tree with `QueryClientProvider` from `@tanstack/react-query`.
