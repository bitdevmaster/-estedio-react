## Main factories

### createResources

```ts
function createResources<TResourceMap>(
  resources: Record<keyof TResourceMap, ResourceConfig>,
  options?: CreateResourcesOptions,
): TanstackResourceHooks<TResourceMap>;
```

### createStore

Alias of `createResources`.

### CreateResourcesOptions

```ts
interface CreateResourcesOptions {
  apiPort?: IApiPort;
  storageByResource?: Partial<Record<string, IStoragePort>>;
}
```

## Provider

### TanstackProvider

Wraps `QueryClientProvider` and optionally calls `initializeLibConfig` on first render.

## Hook factories

### createResourceHooks

Low-level factory used by `createResources` to build typed hooks from a resource registry and adapter dependencies.

### createHooks

Alias of `createResourceHooks`.

## Hooks

### useResourceQuery

Accepts:

- `resource`
- `payload`
- `enabled`
- `force`
- `queryKey`

Returns the normal `ResourceState<T>` shape plus the underlying TanStack Query result fields.

### useResourceMutation

Returns the mutation result plus:

```ts
dispatch(
  payloadOrCallback?: TPayload | FetchCallback<TResponse>,
  callbackArg?: FetchCallback<TResponse>,
): void;
```

## Types

### TanstackResourceHooks

Typed container for `useResourceQuery` and `useResourceMutation`.

### QueryResourceResult

TanStack Query `UseQueryResult` merged with `ResourceState<T>`.

### MutationResourceResult

TanStack Query `UseMutationResult` merged with `ResourceState<T>` and `dispatch`.

### FetchCallback

Shared callback signature used by query and mutation dispatch flows.

## Behavior notes

- `meta.persist` and `meta.persistKey` hydrate the query cache from encrypted storage
- `meta.isAuthResource` persists token responses with the shared token manager
- `sagaEffect` is ignored because it is Redux-specific
