## Main factories

### createResources

```ts
function createResources<TResourceMap>(
  resources: Record<keyof TResourceMap, ResourceConfig>,
  options?: CreateResourcesOptions,
): {
  makeStore: () => EnhancedStore;
  slices: Record<keyof TResourceMap, ResourceSlice>;
};
```

### createStore

Alias of `createResources`.

### CreateResourcesOptions

```ts
interface CreateResourcesOptions {
  apiPort?: IApiPort;
  storageByResource?: Partial<Record<string, IStoragePort>>;
}
```

## Provider

### StoreProvider

```tsx
<StoreProvider makeStore={makeStore} provider={providerConfig}>
  {children}
</StoreProvider>
```

Initializes shared config once and mounts the generated Redux store.

## Hooks

### createResourceHooks

```ts
function createResourceHooks<TResourceMap>(
  slices: Record<keyof TResourceMap, ResourceSlice>,
): {
  useResourceDispatch: () => unknown;
  useResourceSelector: <K extends keyof TResourceMap>(
    resource: K,
  ) => ResourceState<TResourceMap[K]["response"]>;
};
```

### createHooks

Alias of `createResourceHooks`.

### useResourceDispatch

```ts
const dispatch = useResourceDispatch();
dispatch({ resource: "GetUsers", force: false });
dispatch({ resource: "GetUserById" }, { userId: 42 });
```

### useResourceSelector

```ts
const { result, loading, error } = useResourceSelector("GetUsers");
```

## Low-level exports

### createResourceSlice

Creates a Redux Toolkit slice with `fetch`, `fetchSuccess`, `fetchFailure`, `clearResult`, and `reset` reducers.

### createResourceSaga

Creates the per-resource watcher/worker saga used by `createResources`. It applies the shared cache-skip, auth-token, and persistence behaviors.

### assembleReduxStore

Low-level store assembly helper used internally to combine generated reducers and run the root saga.

## Types

### RootStateFromResources

Infers the adapter state shape from the resource map.

### ReduxResourceConfig

Self-documenting alias for Redux-specific resource maps. Equivalent to `ResourceConfig`.

### FetchPayload

Internal saga payload type with `payload`, `force`, and `callback`.

### RootSagaDependencies

Shared dependency shape for advanced saga composition.
