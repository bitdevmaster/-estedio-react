## Main factories

### createResources

```ts
function createResources<TResourceMap>(
  resources: Record<keyof TResourceMap, ResourceConfig>,
  options?: CreateResourcesOptions,
): {
  stores: ResourceStoresFromTypeMap<TResourceMap>;
};
```

### createStore

Alias of `createResources`.

### CreateResourcesOptions

```ts
interface CreateResourcesOptions {
  apiPort?: IApiPort;
  storageByResource?: Partial<Record<string, IStoragePort>>;
}
```

## Provider

### ZustandProvider

Optional wrapper that initializes shared library config once and renders its children unchanged.

## Hooks

### createResourceHooks

Creates typed `useResourceDispatch` and `useResourceSelector` hooks from the generated stores map.

### createHooks

Alias of `createResourceHooks`.

### useResourceDispatch

```ts
const dispatch = useResourceDispatch();
dispatch({ resource: "GetUsers", force: false });
dispatch({ resource: "GetUserById" }, { userId: 42 });
```

### useResourceSelector

```ts
const { result, loading, error } = useResourceSelector("GetUsers");
```

## Low-level exports

### createResourceStore

Creates one vanilla Zustand store per resource and applies shared cache, auth-token, and persistence behavior.

### ResourceStoreState

Includes:

- `result`
- `error`
- `loading`
- `fetch`
- `clearResult`
- `reset`

## Types

### ResourceStoresFromTypeMap

Infers the generated stores map from your resource type map.

### FetchCallback

Shared callback signature used by store fetch operations.
