## Installation

```bash
npm install @estedio/react axios @tanstack/react-query
```

## Setup

The TanStack Query adapter returns typed `useResourceQuery` and `useResourceMutation` hooks backed by your resource definitions.

### 1. Create hooks

```ts
import {
  createStore,
  localStorageAdapter,
} from "@estedio/react/tanstack-query";
import { resources } from "./resources";
import type { ResourceTypeMap } from "./resources";

export const { useResourceQuery, useResourceMutation } =
  createStore<ResourceTypeMap>(resources, {
    storageByResource: {
      GetUsers: localStorageAdapter,
    },
  });
```

`createResources` and `createStore` are equivalent.

### 2. Add a provider

You can keep using your own `QueryClientProvider`, or use `TanstackProvider` to initialize both the `QueryClient` and the shared library config in one place.

```tsx
"use client";

import type { ReactNode } from "react";
import { QueryClient } from "@tanstack/react-query";
import { TanstackProvider } from "@estedio/react/tanstack-query";

const queryClient = new QueryClient();

export function AppProvider({ children }: { children: ReactNode }) {
  return (
    <TanstackProvider
      queryClient={queryClient}
      provider={{ apiBaseUrl: "https://api.example.com" }}
    >
      {children}
    </TanstackProvider>
  );
}
```

### 3. Fetch data with `useResourceQuery`

```tsx
"use client";

import { useResourceQuery } from "./resources";

export function UserList() {
  const { result, loading, error } = useResourceQuery({
    resource: "GetUsers",
    force: false,
  });

  if (loading) return <p>Loading...</p>;
  if (error) return <p>Error: {error}</p>;

  return (
    <ul>
      {result?.map((user) => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  );
}
```

### 4. Mutate data with `useResourceMutation`

```tsx
"use client";

import { useResourceMutation } from "./resources";

export function CreateUserButton() {
  const createUser = useResourceMutation("CreateUser");

  return (
    <button
      onClick={() =>
        createUser.dispatch({ name: "Alice", email: "alice@example.com" })
      }
    >
      Create user
    </button>
  );
}
```

## Query behavior

`useResourceQuery` accepts:

- `resource`
- `payload`
- `enabled`
- `force`
- `queryKey`

When `force` is `false`, the hook sets `staleTime: Infinity` unless you override it, which makes the existing query cache behave like a durable resource hit.

## Persistence and auth

The current implementation supports the same resource metadata used by the Redux and Zustand adapters:

- `meta.persist` and `meta.persistKey` hydrate the query cache from encrypted storage
- `meta.isAuthResource` saves auth token responses through the shared token manager
- `sagaEffect` is still ignored because it is Redux-specific

## Path parameters

Path parameter interpolation works the same as every other adapter:

```ts
const { result } = useResourceQuery({
  resource: "GetUserById",
  payload: { userId: 42 },
  force: false,
});
```
