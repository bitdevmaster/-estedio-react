## What is @estedio/react?

`@estedio/react` gives React applications a single resource model that can be wired into Redux, Zustand, or TanStack Query without changing the resource definitions themselves.

You define resources once, choose an adapter entry point, and get back typed hooks or stores powered by a shared HTTP, token, and persistence runtime.

## Core ideas

### Resources are configuration, not boilerplate

```ts
const GetUsers: ResourceConfig<void, User[]> = {
  endpoint: "/users",
  method: "GET",
  authenticated: true,
};
```

The same resource definition can power:

- Redux slices and sagas
- Zustand resource stores
- TanStack Query queries and mutations

### Shared runtime, adapter-specific state

| Layer           | Responsibility                                                   |
| --------------- | ---------------------------------------------------------------- |
| Core            | `ResourceConfig`, `ResourceState`, `NormalizedApiError`, ports   |
| Infrastructure  | `apiClient`, `ResourceAdapter`, token manager, encrypted storage |
| State behaviors | Cache skipping, token persistence, persisted result hydration    |
| Adapters        | Redux, Zustand, and TanStack Query integration points            |

### One state shape across all adapters

```ts
type ResourceState<T> = {
  result: T | null;
  loading: boolean;
  error: string | null;
};
```

## Adapter comparison

| Feature              | Redux                                | Zustand                              | TanStack Query                                       |
| -------------------- | ------------------------------------ | ------------------------------------ | ---------------------------------------------------- |
| Import path          | `@estedio/react/redux`               | `@estedio/react/zustand`             | `@estedio/react/tanstack-query`                      |
| State model          | Redux Toolkit slices                 | Vanilla Zustand stores               | Query and mutation hooks                             |
| Side-effects         | Redux Saga                           | Internal `async/await`               | TanStack Query runtime                               |
| Provider story       | `StoreProvider`                      | Optional `ZustandProvider`           | `TanstackProvider` or your own `QueryClientProvider` |
| Persistence          | `meta.persist` + `storageByResource` | `meta.persist` + `storageByResource` | `meta.persist` + `storageByResource`                 |
| Auth token auto-save | `meta.isAuthResource`                | `meta.isAuthResource`                | `meta.isAuthResource`                                |
| Best fit             | Complex orchestration                | Lightweight local resource state     | Query-driven server state                            |
